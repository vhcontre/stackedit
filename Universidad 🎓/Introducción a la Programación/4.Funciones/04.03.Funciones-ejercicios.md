### **Ejercicio 15: Encontrar el n√∫mero mayor en una secuencia**

```python
def numero_mayor():
    mayor = float("-inf")
    while True:
        num = int(input("Ingrese un n√∫mero (0 para terminar): "))
        if num == 0:
            break
        if num > mayor:
            mayor = num
    return mayor if mayor != float("-inf") else None

# Prueba
print(numero_mayor())

```

---

### üß† ¬øQu√© hace este programa?
Este programa pide al usuario que ingrese n√∫meros uno por uno. El usuario debe ingresar `0` cuando quiera terminar. El programa devuelve el n√∫mero **m√°s grande** que se ingres√≥.

---

### üí° An√°lisis paso a paso:

```python
def numero_mayor():
```
üü¢ Se define una funci√≥n llamada `numero_mayor`. No recibe par√°metros, todo se hace dentro.

---

```python
    mayor = float("-inf")
```
üîµ Se inicializa la variable `mayor` con `float("-inf")`, que representa **menos infinito**.  
Es una forma de asegurarse de que **cualquier n√∫mero que se ingrese va a ser mayor** que ese valor al principio.

üî∏ En versiones m√°s simples podr√≠as iniciar `mayor = 0` o con una bandera, pero esto es una pr√°ctica com√∫n cuando no se sabe cu√°l ser√° el primer n√∫mero.

---

```python
    while True:
```
üîÅ Bucle infinito: va a seguir pidiendo n√∫meros hasta que se escriba `0`.

---

```python
        num = int(input("Ingrese un n√∫mero (0 para terminar): "))
```
üìù Pide al usuario que ingrese un n√∫mero entero. Si se ingresa `0`, se va a terminar el bucle.

---

```python
        if num == 0:
            break
```
üö™ Si el n√∫mero es `0`, el bucle se corta con `break`.

---

```python
        if num > mayor:
            mayor = num
```
üîç Si el n√∫mero ingresado es mayor al actual guardado en `mayor`, se actualiza.

---

```python
    return mayor if mayor != float("-inf") else None
```
‚úÖ Cuando termina el bucle, se devuelve el valor de `mayor`, **a menos que no se haya ingresado ning√∫n n√∫mero**.

üü° Si no se ingres√≥ ning√∫n n√∫mero (es decir, el usuario escribi√≥ `0` como primer valor), entonces `mayor` sigue siendo `-inf`. En ese caso, devuelve `None`.

---

```python
print(numero_mayor())
```
üß™ Llamado a la funci√≥n y muestra en pantalla el resultado.

### üß© Versi√≥n m√°s simple:
Si a√∫n no se vi√≥ `float("-inf")`, podemos hacer algo semejante:

```python
def numero_mayor():
    primer = True
    while True:
        num = int(input("Ingrese un n√∫mero (0 para terminar): "))
        if num == 0:
            break
        if primer:
            mayor = num
            primer = False
        elif num > mayor:
            mayor = num
    if not primer:
        return mayor
    else:
        return None
```
